These are instructions for creating design philosophies — aesthetic movements that are then EXPRESSED VISUALLY. Output only .md files, .pdf files, and .png files.

Complete this in two steps:
1. Design Philosophy Creation (.md file)
2. Express by creating it on a canvas (.pdf file or .png file)

## DESIGN PHILOSOPHY CREATION

Create a VISUAL PHILOSOPHY (not layouts or templates) interpreted through form, space, color, composition, images, graphics, shapes, patterns, and minimal text as visual accent.

### THE CRITICAL UNDERSTANDING
- What is received: Subtle input from the user — use as foundation, not constraint on creative freedom.
- What is created: A design philosophy/aesthetic movement.
- What happens next: The philosophy is EXPRESSED VISUALLY — artifacts that are 90% visual design, 10% essential text.

### HOW TO GENERATE A VISUAL PHILOSOPHY

**Name the movement** (1-2 words): "Brutalist Joy" / "Chromatic Silence" / "Metabolist Dreams"

**Articulate the philosophy** (4-6 paragraphs) expressing how it manifests through space/form, color/material, scale/rhythm, composition/balance, and visual hierarchy.

**CRITICAL GUIDELINES:**
- **Avoid redundancy**: Each design aspect mentioned once.
- **Emphasize craftsmanship REPEATEDLY**: Stress that the final work must appear meticulously crafted, the product of deep expertise, painstaking attention, master-level execution.
- **Leave creative space**: Specific about aesthetic direction, concise enough for interpretive choices at an extremely high level of craftsmanship.

### PHILOSOPHY EXAMPLES

**"Concrete Poetry"** — Communication through monumental form and bold geometry. Massive color blocks, sculptural typography, Brutalist spatial divisions, Polish poster energy meets Le Corbusier. Text as rare, powerful gesture — never paragraphs. Expert-level precision in every placement.

**"Chromatic Language"** — Color as the primary information system. Geometric precision where color zones create meaning. Minimal typography — small sans-serif labels letting chromatic fields communicate. Josef Albers meets data visualization. Painstaking chromatic calibration.

**"Analog Meditation"** — Quiet visual contemplation through texture and breathing room. Paper grain, ink bleeds, vast negative space. Japanese photobook aesthetic. Images breathe across pages. Each composition balanced with the care of a meditation practice.

**The design philosophy should be 4-6 paragraphs.** Output as a .md file.

## DEDUCING THE SUBTLE REFERENCE

Before creating the canvas, identify the subtle conceptual thread from the original request. The topic is a **subtle, niche reference embedded within the art itself** — not literal, always sophisticated. Someone familiar should feel it intuitively; others simply experience a masterful abstract composition. Think like a jazz musician quoting another song — only those who know will catch it.

## CANVAS CREATION

With philosophy and conceptual framework established, express it on a canvas. Create one single page, highly visual, design-forward PDF or PNG output (unless asked for more pages).

Use repeating patterns, perfect shapes. Treat the design as if it were a scientific bible — dense accumulation of marks, repeated elements, layered patterns that build meaning through patient repetition. Add sparse, clinical typography and systematic reference markers. Anchor with simple phrases positioned subtly, using a limited color palette.

**Text**: Always minimal and visual-first. Let context guide whether that means whisper-quiet labels or bold typographic gestures. Most of the time, font should be thin. All fonts must be design-forward. **Search the `./canvas-fonts` directory for available fonts.** Nothing falls off the page, nothing overlaps. Every element must be contained within canvas boundaries with proper margins and breathing room.

**CRITICAL**: Create work that looks like it took countless hours. Everything must scream expert-level craftsmanship. Double-check that nothing overlaps, formatting is flawless, every detail perfect.

Output the final result as a single .pdf or .png file, alongside the design philosophy as a .md file.

## FINAL STEP — REFINEMENT

**The user ALREADY said "It isn't perfect enough."** Take a second pass. Go back to the code and refine/polish to make this a philosophically designed masterpiece. Avoid adding more graphics — instead refine what exists. Ask: "How can I make what's already here more of a piece of art?"

## MULTI-PAGE OPTION

When requested, create additional pages as unique twists on the original philosophy. Bundle in the same .pdf or many .pngs. Treat pages like a coffee table book — each page tells part of the story with full creative freedom.

## Common Issues

- **Overlapping elements**: The most common mistake. Every element needs breathing room. Check margins, spacing, and layering before finalizing.
- **Too much text**: This is visual art, not a document. If there are more than 10-15 words on the canvas, there's probably too much text.
- **Font not found**: Search `./canvas-fonts` for available fonts. If a desired font isn't available, choose a similar one from what's there.
- **Output looks AI-generated**: The refinement step exists to counter this. Strip away anything formulaic. Real design has intentional imperfection and hierarchy, not symmetric template layouts.
- **Canvas too busy**: When in doubt, remove elements. The philosophy should guide toward restraint, not accumulation.
