## 5. Cross-Surface Mental Model

On Friday evening, a returning farmer likely lands past `/` because authenticated users are redirected (`app/page.tsx:15-17`). `/overview` gives a compressed answer first: one AI stance board across Canadian and US markets (`app/(dashboard)/overview/page.tsx:28-42`). From there, the obvious deeper path is a grain page, but `/grain/[slug]` is gated by whether that grain is in My Farm (`app/(dashboard)/grain/[slug]/page.tsx:91-105`). If unlocked, the farmer gets a thesis, grain-scoped Bushy chat, and farm progress in that order (`app/(dashboard)/grain/[slug]/page.tsx:242-291`). If they bounce, it is probably at the lock screen or at the US surfaces: `/us` and `/us/[market]` use desk terms like `thesis`, `futures`, `confidence`, and `data freshness` (`app/(dashboard)/us/page.tsx:53-70`, `app/(dashboard)/us/[market]/page.tsx:78-144`). `/my-farm` is the practical anchor because it owns crop setup, weekly summary, recommendations, and delivery pace (`app/(dashboard)/my-farm/page.tsx:199-285`).

## 7. Where /seeding Should Plug In

Inventory-based fit: `/seeding` should plug in closer to `/my-farm` than `/overview`. My Farm is the existing farm-owned workspace: it owns crop plans, setup state, recommendations, and delivery pace (`app/(dashboard)/my-farm/page.tsx:199-285`), and `MyFarmClient` already accepts grain/acres-style setup prefill from query params (`app/(dashboard)/my-farm/client.tsx:88-122`). A seeding route should adopt `SectionHeader` for top-level sections because that is the shared dashboard divider (`components/dashboard/section-header.tsx:7-17`) and should adopt `SectionBoundary` where source data or form modules can fail because that is the documented section-isolation pattern (`components/dashboard/CLAUDE.md:5`, `components/dashboard/section-boundary.tsx:13-23`). It should use `GlassCard` when presenting dashboard cards, since the current dashboard card system centralizes elevation, hover, glow, and reduced-motion behavior there (`components/ui/glass-card.tsx:28-67`).

Justifiable break: `/seeding` should not inherit `/overview`'s current one-section stance layout because that surface is a market scan, not a farm setup workflow (`app/(dashboard)/overview/page.tsx:26-53`). It can also break the strict 3-section grain-detail pattern if the job is a seasonal workflow rather than a weekly market read; the audited precedent is `/my-farm`, which already uses five farm-work sections (`app/(dashboard)/my-farm/page.tsx:201-277`) while grain detail keeps thesis/chat/farm as its three-section lane (`app/(dashboard)/grain/[slug]/page.tsx:242-291`).
