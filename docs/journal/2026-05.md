### Lessons learned (folded into AGENTS.md guardrails)

- **Anchor folder-level ignore rules with a leading `/`.** Unanchored `infographic/` could match a hypothetical `app/components/infographic/` six months from now. Codex caught this; Claude missed it.
- **Verify global skill destination before deleting project copies.** The session-skill list at start said `baoyu-imagine` was "available" — but that was because the harness scanned the project's `.claude/skills/`, not because it lived globally. Promotion meant *actually copying* up first, then re-running a session-start skill scan to confirm the global pickup, *then* deleting the local source.
- **Pre-existing tracked junk is invisible until you grep for it.** `~$gsw-shg-31-en.xlsx` and a 135K-line `.bak` were tracked for weeks before Codex flagged them.
- **Rules-only AGENTS.md / CLAUDE.md is the discipline.** The single biggest source of drift in this pass was AGENTS.md slowly accumulating "Phase: MVP + Intelligence" / "(15 tasks)" / "Implementation Plans" bullets. New convention: any time CLAUDE.md is updated, check whether AGENTS.md has the same status content and prune it.
